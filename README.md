# Персональная база знаний для подготовки к техническим собеседованиям

Веб-приложение для личного использования: хранение технических вопросов и ответов, повторение карточек и подсветка слабых мест.

## Что умеет приложение

- добавление вопросов и ответов (профессия / тема / подтема выбираются из существующих или вводятся новые)
- редактирование и удаление карточек
- фильтрация по профессии
- режим повторения сессиями: выбираешь профессию и сколько карточек (10 / 20 / 30 / все)
- оценки ответа: `Знаю`, `Не уверен`, `Не знаю`
- у каждой карточки есть уровень освоения 0–5, слабые карточки попадаются чаще
- экспорт и импорт всей базы в JSON (бэкап в один файл)
- адаптивный интерфейс: на телефоне — нижняя навигация и выезжающие панели
- тёмный минималистичный дизайн
- PWA: ставится на домашний экран iPhone/Android иконкой и открывается в полноэкранном режиме

## Как работает повторение

Никаких интервальных повторений по датам — повторять можно в любой момент.

Вместо этого у каждой карточки есть **уровень освоения от 0 до 5**:
- `Знаю` → уровень +1 (максимум 5)
- `Не уверен` → уровень −1 (минимум 0)
- `Не знаю` → уровень сбрасывается в 0
- новая карточка начинается с уровня 0

При сборе сессии карточка получает вес `6 − уровень`: уровень 0 попадает в выборку
в 6 раз чаще, чем уровень 5 (но выученное тоже иногда повторяется). Внутри одной
сессии карточки не дублируются. Прогресс по карточке можно сбросить вручную в её деталях.

## Установка на iPhone (PWA)

Приложение — устанавливаемое PWA. Чтобы повесить иконку на домашний экран айфона:

1. Открой приложение в **Safari** (именно Safari — Chrome на iOS не умеет установку).
2. Нажми «Поделиться» → «На экран „Домой“».
3. Готово — иконка на экране, открывается без адресной строки, как нативное приложение.

Важно: для установки сайт должен отдаваться по **HTTPS** (исключение — `localhost`). По голому
`http://` с локального IP установка/полноэкранный режим на iOS не активируются. Самый простой
способ получить HTTPS для личного доступа — Tailscale (MagicDNS) или reverse-proxy с Let's Encrypt.

Данные карточек живут на сервере (SQLite через API), поэтому без сети приложение работать не будет
— это «онлайн-PWA», а не полностью офлайновое.

## Хранение данных

Данные хранятся в SQLite-базе — это один файл, что удобно для бэкапа.

В Docker-режиме база лежит в volume `kb_data` (`/app/data/dev.db`, путь задаётся через
`DATABASE_URL` в `docker-compose.yml`) и сохраняется между перезапусками контейнера.

### Бэкап и перенос на другой сервер

Самый простой способ не потерять вопросы — встроенный экспорт/импорт:
- **Скачать базу** — выгружает все карточки в JSON-файл к тебе на компьютер.
- **Загрузить базу** — заливает JSON обратно (обновляет существующие карточки по `id`,
  добавляет новые). Так базу можно перенести на новый сервер или поднять локально.

Альтернатива — забрать файл базы напрямую:

```bash
docker compose cp app:/app/data/dev.db ./dev.db
```

## Локальный запуск

1. Установить зависимости:

```bash
npm install
```

2. Сгенерировать Prisma Client:

```bash
npm run db:generate
```

3. Применить схему базы:

```bash
npx prisma db push
```

4. Запустить проект:

```bash
npm run dev
```

Приложение будет доступно по адресу:

```text
http://localhost:3000
```

## Запуск через Docker

1. Убедиться, что установлен Docker и Docker Compose.

2. Создать `.env` на основе шаблона:

```bash
cp .env.example .env
```

3. Запустить проект:

```bash
docker compose up -d --build
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:3000
```

## Структура проекта

- `app/page.tsx` — серверный компонент: грузит карточки из БД при заходе
- `app/knowledge-base.tsx` — клиентский интерфейс (база, повторение, формы)
- `app/api/questions/` — API для вопросов (CRUD + оценки/сброс уровня)
- `app/api/export/` — выгрузка базы в JSON
- `app/api/import/` — загрузка базы из JSON
- `lib/prisma.ts` — Prisma client
- `lib/validation.ts` — валидация данных
- `prisma/schema.prisma` — схема базы данных
- `Dockerfile` — сборка контейнера
- `docker-compose.yml` — запуск на сервере
