# Персональная база знаний для подготовки к техническим собеседованиям

Веб-приложение для личного использования: хранение технических вопросов и ответов, повторение карточек и подсветка слабых мест.

## Что умеет приложение

- добавление вопросов и ответов
- редактирование и удаление карточек
- фильтрация по профессии
- режим повторения вопросов
- оценки ответа:
  - `Знаю`
  - `Не уверен`
  - `Не знаю`
- слабые карточки показываются чаще
- адаптивный интерфейс для телефона, планшета и десктопа
- тёмный минималистичный дизайн

## Как работает повторение

В приложении нет жёсткого интервального повторения по датам.

Вместо этого:
- повторять вопросы можно в любой момент
- карточки, отмеченные как `Не уверен` и `Не знаю`, получают больший вес
- такие вопросы чаще попадают в следующую сессию повторения

Это удобно для личной подготовки без сложной настройки алгоритма.

## Технологии

- Next.js 16
- TypeScript
- Tailwind CSS
- Prisma
- SQLite
- Docker / Docker Compose

## Хранение данных

Данные хранятся в SQLite-базе.

В Docker-режиме база лежит в volume и сохраняется между перезапусками контейнера.

## Локальный запуск

1. Установить зависимости:

```bash
npm install
```

2. Сгенерировать Prisma Client:

```bash
npm run db:generate
```

3. Применить схему базы:

```bash
npx prisma db push
```

4. Запустить проект:

```bash
npm run dev
```

Приложение будет доступно по адресу:

```text
http://localhost:3000
```

## Запуск через Docker

1. Убедиться, что установлен Docker и Docker Compose.

2. Создать `.env` на основе шаблона:

```bash
cp .env.example .env
```

3. Запустить проект:

```bash
docker compose up -d --build
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:3000
```

## Структура проекта

- `app/page.tsx` — основной интерфейс приложения
- `app/api/questions/` — API для вопросов
- `lib/prisma.ts` — Prisma client
- `lib/validation.ts` — валидация данных
- `prisma/schema.prisma` — схема базы данных
- `Dockerfile` — сборка контейнера
- `docker-compose.yml` — запуск на сервере
